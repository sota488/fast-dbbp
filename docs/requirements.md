# Fast Fold Omaha 練習サイト 要件定義

## 1. プロジェクト概要
Fast Fold Omaha Practiceは、ブラウザ上で動作するPlay Money専用のPot Limit Omaha練習サイトです。6-maxテーブル形式で、Fast Foldの仕組みを使い、プレイヤーが手早く多くのハンドを学べる体験を提供します。将来的にはBot対戦にも対応し、練習用AIや対戦相手としてのBotを追加できる設計にします。

## 2. 必要な技術一覧
### フロントエンド
- HTML / CSS
- JavaScript / TypeScript
- UIライブラリ（例: React, Vue, または Svelte）
- 状態管理（例: Redux, Zustand, Vuex, Piniaなど）
- CSSフレームワーク（例: Tailwind CSS, Bootstrap, Material UI）

### ゲームロジック
- ポーカーゲームのルール実装
- Pot Limit Omahaのベッティングルール
- Fast Foldテーブル切り替えロジック
- 6-maxテーブルのプレイヤー管理
- 手札評価ロジック（Omahaの5カードベストハンド判定）

### バックエンド（将来オプション）
- Node.js / Deno / Cloud Functions
- API サーバー（ゲーム履歴、プレイヤーデータ、Bot管理）
- WebSocket / Socket.IO（リアルタイム対戦用）
- データベース（Play Money残高、統計、Bot設定）

### Bot対戦対応（将来）
- Bot API / AIエンジン実装
- Bot用状態管理と行動選択ロジック
- ルールベースBot / 機械学習Botの拡張性
- Botと人間プレイヤーの混在管理

### 開発環境・ツール
- Git / GitHubなどのバージョン管理
- パッケージマネージャー（npm / yarn / pnpm）
- テストフレームワーク（Jest, Vitestなど）
- Linter / Formatter（ESLint, Prettier）
- ドキュメント管理（Markdown）

## 3. 開発ロードマップ
### フェーズ1: 企画と設計
- ゲームの主要要件を整理する
- Pot Limit Omahaのルールを明確化する
- Fast Foldのユーザー体験を定義する
- 画面構成と操作フローを設計する

### フェーズ2: 基本プロトタイプ
- 1テーブルの6-maxゲームを実装する
- ディーラー・プリフロップ・フロップ・ターン・リバーの流れを実装
- Pot Limit Omahaのベッティングとフォールド・チェック・コールを実装
- Play Money残高とブラインドを管理する
- シンプルなブラウザUIを構築する

### フェーズ3: Fast Fold対応
- プレイヤーがフォールドしたら別テーブル／別ハンドに即座に移動する仕組みを設計
- テーブルプールを実装して、プレイヤーが常に新しいハンドで遊べるようにする
- Fast Foldの切り替えアニメーション・通知を追加

### フェーズ4: UI/UX改善と拡張
- テーブル表示を見やすくする
- ハンド履歴と統計を追加する
- プレイヤーのアバター・チャット・アクションログを追加
- ゲームルームや複数テーブルに対応する設計を検討

### フェーズ5: Bot対戦準備
- Botインターフェースを設計する
- Botの行動選択ルールを実装する（例: レイズ幅、ポジション、ポットオッズ）
- Botとの対戦モードを追加する
- 将来的に機械学習や強化学習Botを統合できる拡張性を保つ

## 4. 機能要件
- プレイヤーは6人までのテーブルに参加できる
- ゲームはPot Limit Omahaで行われる
- プレイヤーは手札4枚から2枚、ボード5枚から3枚を組み合わせて最良の5枚を作る
- Fast Foldではフォールド後に即座に新しいテーブルまたはハンドに移動できる
- Play Moneyのみで実際の賭けはなし
- ブラウザ上で動作し、インストール不要

## 5. 非機能要件
- 将来的にBot対戦を容易に追加可能なコード構造
- ルールエンジンとUIは分離された設計
- わかりやすい操作性と高速なレスポンス
- フロントエンド中心のアーキテクチャで、必要に応じてバックエンド追加が可能
- セキュリティ面ではリアルマネーを扱わないため、ユーザーのプライバシー重視

- レベルやトレーニング課題の追加
- ハンド履歴分析とプレイヤー統計
- ユーザーアカウントとランキング機能
